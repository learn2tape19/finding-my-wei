## The Decision

Atlas 01's primary contribution is:

**An Original Clinical Reasoning Framework**

Not a biological discovery.  
Not a theoretical innovation.  
Not a terminology invention.

A coherent, evidence-informed model for how clinicians reason about creating therapeutic change in the presence of another nervous system.

---

## What This Means

### What We Are Building

A reasoning framework that helps clinicians answer:

> **How do we intentionally create the conditions that make therapeutic change more likely?**

The framework integrates:
- Autonomic regulation (nervous system state)
- Permission (safety + autonomy)
- Contextual effects (meaning + expectation)
- Prediction updating (learning capacity)
- Therapeutic alliance (relational safety)
- Clinical communication (explicit + implicit)
- Sensorimotor engagement (embodied learning)

The contribution is **how these are organized into a coherent reasoning model**, not the components themselves.

### What We Are NOT Building

❌ A new biological mechanism  
❌ A theoretical replacement for existing models  
❌ A new clinical technique or protocol  
❌ A standalone concept (Permission, nervous system safety, etc.)  
❌ A research hypothesis to be tested  

### What This Enables

✓ Clinicians can reason about their work with greater coherence  
✓ Teaching programs can organize curriculum around core principles  
✓ Future research can test specific components of the framework  
✓ Clinical decision-making becomes more intentional  
✓ Different manual therapy approaches can be evaluated through the framework lens

## The Core Research Question

What is the minimum set of principles required for a clinician to reason effectively in the presence of another nervous system?

### This Question Guides:
- What concepts are included in the framework (only essential principles)
- How concepts are organized (logical flow for reasoning)
- What gets emphasized (what clinicians most need to understand)
- What gets de-emphasized (what is secondary)

- How the framework is tested (do these principles improve reasoning?)

## Framework Architecture

The clinical reasoning framework should answer these sequential questions:

### 1. Assessment
> What is the nervous system state I'm encountering?

Principles:
- Nervous system regulation is foundational to all else
- Client nervous system state is observable and interpretable
- Your own nervous system state affects what the client's system perceives

### 2. Foundation
> What conditions must be established before intervention?

Principles:
- Safety (nervous system must perceive non-threat)
- Permission (client must feel agency to decline)
- Clarity (client must understand what and why)
- Expectation (meaning shapes response)

### 3. Engagement
> How do I create conditions for learning to happen?

Principles:
- Prediction updating requires safety
- Embodied learning happens through touch + nervous system + attention
- Clinician regulation enables client regulation
- Alliance is the container for change

### 4. Integration
> How does change consolidate and persist?

Principles:
- Nervous system learning requires felt experience, not just information
- Permission to explore (and to decline) enables sustainable change
- Context and expectation continue shaping interpretation
- Autonomy preservation enables ownership of change

### 5. Reflection
> What did the nervous system learn? What should inform future sessions?

Principles:
- Track shifts in what client can perceive and do
- Notice what supported change (was it nervous system? Understanding? Expectation? All?)
- Adjust framework based on individual client nervous system
- Build evidence for what this particular nervous system responds to

---

## What Makes This Framework Original?

Not any individual principle (all are evidence-supported).

**The originality is in:**

1. **Organization**: Sequential reasoning from assessment → foundation → engagement → integration → reflection
2. **Application**: Specific to brief-encounter manual therapy (not 50-week psychotherapy)
3. **Centrality of nervous system**: Makes autonomic regulation the primary lens for all other decisions
4. **Permission integration**: Places autonomy/safety at foundation level (not peripheral ethical requirement)
5. **Minimum viable set**: Identifies what clinicians absolutely need to understand vs. optional detail

## From Components to Framework

### Before (Component-Focused)
- Alliance is important
- Permission is important
- Nervous system regulation is important
- Context matters
- Prediction updating enables learning

**Problem**: Components don't connect; they feel like separate things

### After (Framework-Focused)
- Assessment: What nervous system am I with?
- Foundation: How do I create safety + permission + clarity + expectation?
- Engagement: How do I enable prediction updating through nervous system regulation?
- Integration: How does embodied learning consolidate?
- Reflection: What did this nervous system learn?

**Benefit**: Components connect into a coherent reasoning model
