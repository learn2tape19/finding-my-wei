# Key Insights Embedded in This Design

## 1. Domains are Primary
Projects are temporary. Domains are permanent. Work belongs to domains, which belong to organizations (entities), which create intellectual assets.

## 2. Knowledge Has States
Work moves through states (creation → preservation), not just folders. Management system, not storage system.

## 3. Capabilities are Shared
Marketing, publishing, research, analytics — these serve all domains and should not be replicated domain-by-domain.

## 4. Self-Description is Essential
The system must explain itself. Every major directory has a CHARTER.md. Future collaborators need no external explanation.

## 5. Governance is Clear
Every decision, every placement, every evolution is governed by charters and constitutional principles. Authority flows from Constitution → Pillars → Domains → Projects.

## 6. Stewardship Matters
Drew stewards. Claude preserves. Repository records. Roles are clear. Responsibilities are documented.
