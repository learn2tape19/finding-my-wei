# How to Use This Layer

## If You're A New Collaborator

Read in this order:
1. **CONSTITUTION.md** — Understand what this system is
2. **STEWARDSHIP_MODEL.md** — Understand how you should work within it
3. **IEMS_FINAL_ARCHITECTURE.md** — Understand where things live
4. **INSTITUTIONAL_PRINCIPLE.md** — Understand why it's organized this way

Then navigate to the appropriate domain or capability CHARTER.md for specific guidance.

## If You're Making A Decision

1. Check the **CONSTITUTION.md** — Is this decision constitutional?
2. Check the **STEWARDSHIP_MODEL.md** — Is this role-appropriate?
3. Check the appropriate **CHARTER.md** — Does this domain/capability govern this decision?
4. Check the **INSTITUTIONAL_PRINCIPLE.md** — Does this improve coherence or introduce novelty?

## If You're Changing Something

**Ask first:** "Does this change the architecture or improve the institution?"

- If it changes architecture → No (architecture is frozen)
- If it improves the institution → Yes (propose it, get approval, document it)

# Amendment Process

These documents can be amended. But rarely. And only through formal process:

1. **Proposed amendment** is presented with clear rationale
2. **Constitutional impact** is assessed (how does this affect the foundation?)
3. **Founder approval** (Drew Freedman decides)
4. **Amendment documented** in AMENDMENTS.md
5. **Version incremented** in the document
6. **Effective date recorded**

---

# What Cannot Change

The following principles are constitutional and cannot be amended:

- Finding My Wei is entity-driven, not project-driven
- Relationships matter more than files
- Knowledge is living, not static
- Stewardship roles are clear and accountable
- Architecture is permanent so stewardship can work
- Future collaborators should understand the system without explanation
- Success is measured by wisdom, not complexity

---

# What Can Evolve

Within the frozen architecture, the institution can evolve:

- New domains (when business demands them)
- New capabilities (when real need emerges)
- Improved stewardship practices (through experience)
- Deeper understanding of how systems work (through use)
- Refined lifecycle definitions (through application)
- Better relationship documentation (through learning)
