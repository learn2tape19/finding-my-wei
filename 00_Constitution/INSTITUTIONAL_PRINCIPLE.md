# The Distinction

**Freeze the architecture, not the institution.**

Finding My Wei has three permanent foundations:

1. **CONSTITUTION.md** — Governance principles (frozen)
2. **STEWARDSHIP_MODEL.md** — How knowledge lives (frozen)  
3. **IEMS Architecture** — Folder structure and charters (frozen)

**Everything else is living.**

Capabilities evolve. Domains grow. Knowledge flows. Thinking deepens.

The institution breathes. The architecture does not.

# The Principle in Practice

## When New Work Arrives

**Question:** Does it fit within the existing domains and capabilities?

**If yes:** Place it in the appropriate domain, tag with lifecycle state, done.

**If no:** Does it require a new domain or capability?

**If new domain:** Propose it, get Drew's approval, create domain CHARTER, add to 05_DOMAINS, establish steward.

**If new capability:** Propose it, get Drew's approval, create capability CHARTER, add to 04_CAPABILITIES, establish steward.

**Never:** Change the architecture itself. Always: Extend within the architecture.

---

## When Architecture Questions Arise

**Question:** "Should we reorganize [something]?"

**Answer:** No. The architecture is frozen.

**Question:** "Can we add [new capability]?"

**Answer:** Yes, if it serves real need and improves coherence.

**Question:** "Can we change [structure]?"

**Answer:** No. Changes improve what exists; they don't replace it.

**Question:** "Is this the best structure?"

**Answer:** It's the right structure because it's permanent and people understand it. Best is less important than stable.

Finding My Wei will not be killed by these things because:

✅ Architecture is frozen (one version of truth)

✅ Stewardship roles are transferable (survives succession)  
✅ Everything explains itself (CHARTERS provide clarity)  
✅ Purpose is clear (wisdom, not complexity)
