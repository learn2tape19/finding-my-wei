# Repository Constitution: Finding My Wei Operating System

The governing document for Finding My Wei as an Intellectual Operating System.

This is not a policy. This is constitutional law. It is binding. It guides all future work, decisions, and collaborations.

---

## Preamble

Finding My Wei exists to **transform experience into wisdom through systematic thinking**.

We do not build a repository to remember. We build a system to **think**.

Every principle, every process, every structure exists to make you **smarter**, not just to preserve what you already know.

This Constitution governs how that happens.

---

## Article I: Purpose

### The Vision
Finding My Wei is an operating system for a lifetime of intellectual work.

It captures, processes, decides, creates, publishes, reflects, learns, and archives.

Every input (conversation, book, observation, failure, success, research) flows through the same system.

Every output is connected to other work and preserved for future reference.

### The Promise
Over time, this system:
- ✓ Transforms experience into wisdom
- ✓ Makes you smarter about how you think
- ✓ Enables compounding (each project improves the next)
- ✓ Creates intellectual wealth (ideas that outlast you)
- ✓ Supports emergence (one insight becomes many forms)

## Article II: Core Values

### Permanence Over Temporary
What enters this system is permanent. We build for decades, not quarters.

**This means**:
- GitHub is the source of truth (outlasts platforms)
- Markdown is the format (outlasts tools)
- Decisions are recorded (not forgotten)
- Nothing is deleted (everything is archived)
- Version control preserves history

### Simplicity Over Complexity
We reduce complexity rather than create it.

**This means**:
- One logical location for every asset
- Clear standards, not ambiguous guidelines
- Refusal to over-engineer
- Elegant systems, not complicated ones
- A user should understand it without external explanation

### Reflection Over Reaction
We extract learning systematically from every experience.

**This means**:
- Daily, weekly, monthly, quarterly, annual reflection
- Principles extracted from patterns
- Lessons integrated into operating system
- Every project improves the next
- Thinking about your thinking

### Creation Over Collection
Ideas are not valuable until they become something.

**This means**:
- Capture feeds into decision and creation
- Ideas become research, papers, lectures, products, principles
- Emergence is the goal, not archiving
- Output is measured, not input
- The system creates, not just preserves

### Scalability Over Optimization
We build for growth without structural redesign.

**This means**:
- The 22-directory structure works for 10 projects or 100
- Processes scale without changing
- New collaborators can onboard without explanation
- Technology choices don't lock us in
- The system grows, it doesn't break

## Article III: The Operating System

Finding My Wei is built on five integrated systems. All five must work together.

### 1. Universal Insight Processor (UIP)
The front door for every idea.

**What it does**: Every input (conversation, book, research, observation, failure, success) flows through a 10-step transformation:

Capture → Decode → Extract Principles → Connect → Decide → Create → Publish → Reflect → Learn & Integrate → Archive

**Why it matters**: Same rigor on every input. Prevents important ideas from getting lost. Teaches systematic thinking.

**Implementation**: [[UNIVERSAL_INSIGHT_PROCESSOR.md]]

---

### 2. Decision Engine
How important decisions get documented and reviewed.

**What it does**: Every decision records Context, Alternatives, Reasoning, Tradeoffs, Outcome, Lessons, and Future Review Date.

**Why it matters**: Decisions compound. Documenting them prevents repetition and builds decision wisdom.

**Implementation**: [[DECISION_ENGINE.md]] in [[04_Decision_Framework/]]

---

### 3. Reflection Engine
How learning gets extracted and integrated.

**What it does**: Daily, weekly, monthly, quarterly, and annual reflection using systematic questions:
- What changed?
- Why did it matter?
- What principle emerged?
- Where else does this apply?
- Should this become research, content, teaching, framework, product, or principle?

**Why it matters**: Reflection is how experience becomes wisdom. Without it, you repeat the same year 20 times.

**Implementation**: [[REFLECTION_ENGINE.md]]

---

### 4. Knowledge Graph
How everything connects.

**What it does**: Documents how Projects, People, Concepts, Principles, Research, Products, and Questions relate to each other.

**Why it matters**: Nothing should exist in isolation. Seeing connections multiplies value. Discovering related ideas becomes possible.

**Implementation**: [[KNOWLEDGE_GRAPH_ARCHITECTURE.md]]

---

### 5. Daily Workflow
How all systems integrate into a working rhythm.

**What it does**: Describes daily, weekly, monthly, quarterly, and annual rhythms:
- Daily: Capture, review, clarify, work, decode, prepare
- Weekly: Reflection, connection, next-week prep
- Monthly: Theme extraction, principle refinement
- Quarterly: Big picture review, strategic planning, OS assessment
- Annual: Complete year review, system overhaul, next-year planning

**Why it matters**: Systems only work if they're lived. The Daily Workflow makes the operating system real.

**Implementation**: [[DAILY_WORKFLOW.md]]

## Article IV: Standards & Discipline

### File Naming
- Folders: `PascalCase_With_Underscores`
- Files: `kebab-case-file-name.md`
- Dates: ISO 8601 format (YYYY-MM-DD)
- Versions: Through git commits, not filename versioning

### Markdown Standards
- One H1 (`#`) per file — the title
- Use H2 (`##`) for major sections
- Link related documents with `[[name]]` syntax
- Keep files focused and modular

### Git Standards
- Format: `[Category] Subject — brief explanation`
- Categories: [Tao], [Learn2Tape], [Sidekick Air], [Finding My Wei], [Research], [Admin]
- Atomic commits (one idea per commit)
- Descriptive messages (future-you will thank you)

### What Belongs in GitHub
- Principles and frameworks
- Important decisions and their rationale
- Project documentation and milestones
- Research and findings
- Teaching materials and lectures
- Published work and links
- Reflections and learnings

### What Doesn't Belong
- Casual chatter or temporary logistics
- Throwaway drafts or unprocessed notes
- Duplicate fragments or fragments
- Proprietary information (unless cleared)
- Client-specific details

## Article VI: Publishing Philosophy

### When to Publish
An idea is ready to publish when:
- It has been processed through the UIP
- Principles have been extracted
- It connects to existing work
- It is written clearly and completely
- It serves the audience (not just yourself)

### Where to Publish
- **GitHub** (always) — Preserve in permanent source of truth
- **Website** (if public-facing) — taoclinicaltouch.com, learn2tape.com, etc.
- **Social** (if shareable) — Through Buffer
- **Academic** (if significant) — Papers and conferences
- **Teaching** (if educable) — Courses and lectures
- **Products** (if market potential) — Courses, books, tools

### Publishing Discipline
- ✓ Clear, not vague
- ✓ Specific examples, not abstractions
- ✓ Timeless, not trendy
- ✓ Complete, not fragments
- ✓ Sourced, not claimed without evidence
- ✓ Actionable, not theoretical

### The Goal
Publish work that will still matter in 5 years. Avoid viral and temporary. Favor evergreen and durable.

## Article VIII: Long-Term Stewardship

### Permanence
This system is designed to outlive any individual project.

It assumes:
- Work will continue for decades
- Principles will evolve but compound
- Collaborators will change
- Technology will change
- The core operating system endures

### Accessibility
Future readers (humans or AI) should understand this system without external explanation.

**This means**:
- Every folder has a README
- Every major document is self-contained
- Linking reveals connections
- The structure is intuitive
- The Constitution explains everything

### Modularity
Folders should work independently while connecting to each other.

A researcher should be able to work in `07_Research/` without understanding `12_Social_Content/`.

A teacher should be able to work in `10_Lectures/` without understanding `13_Media/`.

But everything connects through the Knowledge Graph.

### Evolution
The operating system itself improves over time.

- Daily Workflow evolves based on what works
- Core Principles expand with new learning
- The Constitution is permanent but new articles can be added
- Processes change; philosophy doesn't
