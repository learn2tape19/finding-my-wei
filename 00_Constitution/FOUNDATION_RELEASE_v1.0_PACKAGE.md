## 2. Institutional Boundary Definition
**File:** `00_CONSTITUTION/INSTITUTIONAL_BOUNDARY_DEFINITION.md`  
**Status:** Version 1.0 (Ratified July 5, 2026)  
**Authority:** Drew Freedman  
**Amendments:** 7 (incorporated as Amendment 001)

Establishes institutional boundaries and asset classifications required before any migration or ingestion.

**Five Permanent Classifications:**
1. Constitutional — Governance documents
2. Intellectual — Original work for Intellectual Estate
3. Institutional — Decision history and strategic records
4. Development — Work in progress (not permanent by default)
5. Operational Support — Temporary business materials (usually excluded)

**Key Frameworks:**
- Institutional Lifecycle (Capture → Development → Review → Classification → Approval → Permanent → Archive)
- Graduation Test (5-question evaluation for permanence)
- Repository Health Standards (signal over volume)
- Migration Protection Rule (identify, inventory, index, classify, approve BEFORE moving)
- Repository-First Workflow Rule (GitHub authoritative; surface conflicts to Drew)
- GitHub/Notion/Drive boundaries
- Business asset evaluation framework (Learn2Tape, Tao, SidekickAir, Boston Bodyworker, Founder)

**Permanence:** Constitutional. Amendments require formal process.
