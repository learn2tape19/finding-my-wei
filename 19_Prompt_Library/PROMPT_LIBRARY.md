# Prompt Library

Reusable prompts for common tasks across Finding My Wei. Copy, customize, and use.

---

## Research & Synthesis Prompts

### Deep Research Prompt
```
You are researching [TOPIC] for the Finding My Wei system.

Context: [Describe the business, project, or question]

Research these aspects:
- Current state of the field
- Key players and innovations
- Gaps and opportunities
- Relevant frameworks or case studies
- Contradictions or tensions in the field

Return: Synthesis (not a list) with connections to our existing work. 
Format: Markdown, ready for GitHub.
```

### Literature Review
```
Synthesize the following sources about [TOPIC]:

[Paste sources]

For Finding My Wei, I need:
- Core claims and supporting evidence
- How these ideas relate to our framework of [FRAMEWORK]
- What we should adopt, adapt, or reject
- Gaps in this literature from our perspective

Return: A coherent synthesis with explicit links to our existing work.
```

## Writing & Content Prompts

### Long-form Writing (Papers, Essays, Chapters)
```
I'm writing a [PAPER/ESSAY/BOOK CHAPTER] about [TOPIC] for [AUDIENCE].

Key points I want to convey:
1. [Point]
2. [Point]
3. [Point]

Tone: [professional/conversational/academic/accessible]
Length: [word count]

Please draft a [SECTION/CHAPTER/ESSAY] that:
- Opens with a compelling hook
- Builds a clear argument
- Uses examples and evidence
- Connects to broader implications
- Closes with actionable insight

I'll refine this further. Focus on structure and flow.
```

### Social Media Series
```
I have a core idea: [IDEA]

I want to create a 3-5 post series for [PLATFORM] that:
- Builds from a hook to depth
- Each post stands alone but links to others
- Drives to a specific call-to-action
- Uses [TONE] voice

Target audience: [AUDIENCE]

Please draft the posts with specific copy, hooks, and any visual directions.
```

## Reflection & Analysis Prompts

### Project Postmortem
```
Project: [NAME]
Timeline: [DATES]
Result: [SUCCESS/PARTIAL/FAILED]

What happened:
- What we aimed to achieve
- What actually happened
- Key decisions made
- Unexpected outcomes

Help me extract learning:
- What went well (and why)?
- What didn't (and why)?
- What would we do differently?
- What's the core insight for future work?

Format: Structured reflection document.
```

### Quarterly Reflection
```
Quarter: [Q1/Q2/Q3/Q4] 2026

Projects completed: [List]
Projects in progress: [List]
Major decisions: [List]

Help me:
1. Assess what worked and what didn't
2. Identify emerging patterns
3. Extract principles and frameworks
4. Recommend priorities for next quarter

Return: Markdown document, ~1000 words, honest assessment.
```

## Creative & Strategy Prompts

### Product Positioning
```
Product/Service: [NAME]
Target market: [DESCRIPTION]
What it does: [BRIEF]
Current positioning: [IF ANY]

Help me craft positioning that:
- Articulates the core value clearly
- Differentiates from alternatives
- Resonates with the target audience
- Guides all future marketing and product decisions

Please provide:
1. Core positioning statement (1-2 sentences)
2. Key messages (3-5 bullets)
3. Value proposition
4. Competitive context
```

### Business Model Analysis
```
I'm evaluating a [BUSINESS/PRODUCT/SERVICE] with:
- Revenue model: [DESCRIBE]
- Cost structure: [DESCRIBE]
- Customer acquisition: [DESCRIBE]
- Unit economics: [IF KNOWN]

I want to understand:
1. Is this sustainable long-term?
2. What are the key vulnerabilities?
3. What would make this work better?
4. How does this compare to alternatives?

Please provide honest assessment with reasoning.
```

## GitHub & Documentation Prompts

### README Generator
```
For GitHub repository: [REPO_NAME]

Purpose: [BRIEF DESCRIPTION]
Audience: [WHO USES THIS]
Key features/folders: [LIST]

Create a README that:
- Hooks the reader in first line
- Explains what this is (and what it isn't)
- Shows quick start or key navigation
- Links to detailed docs
- Is clear for both newcomers and experts

Format: Markdown, GitHub-ready.
```

### Architecture Documentation
```
System/Project: [NAME]

Current structure:
- [Component 1]
- [Component 2]
- [Component 3]

Help me document the architecture:
1. High-level overview (diagram in ASCII or description)
2. Component responsibilities
3. Key design decisions and rationale
4. Future considerations

Make it accessible to someone new to the project.
```

## Using This Library

1. **Copy the prompt** that fits your task
2. **Fill in the bracketed sections** with your specific context
3. **Customize tone and length** as needed
4. **Iterate**: First version rarely perfect; refine based on output
5. **Save good variants**: If you create a variation that works well, add it to the library
