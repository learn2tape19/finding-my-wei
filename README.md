# Finding My Wei

## Intellectual Operating System (IOS)

Finding My Wei is a system for capturing, organizing, refining, preserving, and publishing ideas throughout an entire lifetime of creative work.

Unlike traditional knowledge systems, Finding My Wei does not merely store information. It **transforms experience into wisdom**.

Every interaction contributes to a permanent intellectual estate. What was created. Why it was created. How it evolved. What changed. What was learned.

---

## Core Principles

### 1. **Permanence**
Every meaningful idea becomes a permanent asset. Nothing valuable is temporary.

### 2. **Simplicity**
The system reduces complexity rather than create it. There is always one logical location for every asset.

### 3. **Reflection**
Knowledge is not static. Every project should improve future work.

### 4. **Creation**
Every insight is capable of evolving into:
- Research
- Lectures
- Papers
- Books
- Presentations
- Social media
- Products
- Future ideas

### 5. **Scalability**
The architecture supports decades of work without structural redesign.

## How It Works

Every idea follows the same lifecycle:

```
Capture → Research → Insight → Markdown → GitHub → Visual Design → Publishing → Analytics → Reflection → Knowledge Preservation → Future Creation
```

### GitHub: Permanent Source of Truth
All intellectual assets live in this repository. Version control preserves history. Cross-linking connects ideas.

### Claude & ChatGPT: Complementary AI Partners
- **Claude**: Repository architecture, documentation, markdown, knowledge graphs, long-form synthesis
- **ChatGPT**: Strategic thinking, creative exploration, coaching, business development, teaching

### Adobe Express + Firefly: Visual Production
Transforms written ideas into visual assets for all platforms.

### Buffer: Publishing Engine
Centralized scheduling, campaign organization, evergreen recycling, analytics.

### Analytics & Reflection
Every published piece generates insights that feed back into the system.

## Standards

### File Naming
- Use kebab-case for files and folders: `my-document-title.md`
- Include date for time-sensitive work: `2026-03-15-project-update.md`
- Version control through git commits, not filename versioning

### Markdown Standards
- One H1 (`#`) per file — the title
- Use H2 (`##`) for major sections
- Link related documents with `[[name]]` syntax
- Keep files focused and modular

### Git Commits
- Descriptive, atomic commits
- Format: `[Category] Subject — brief explanation`
- Example: `[Tao] Complete CE learning objectives — NCBTMB compliant language`

### What Belongs Here
- Principles, frameworks, and durable insights
- Research, findings, and synthesis
- Project documentation and decisions
- Teaching materials and lectures
- Published work and assets
- Reflections and learnings

### What Doesn't Belong
- Casual chatter or temporary logistics
- Throwaway drafts or unprocessed notes
- Duplicate fragments or fragments
- Unprocessed noise
